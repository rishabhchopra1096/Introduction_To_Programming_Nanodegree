>HTML DOCUMENT STRUCTURE

              1. <!DOCTYPE html>
              2. <html>
              3.        <head>
              4.                <title>My Web Page!</title>
              5.        </head>
              6.        <body>
              7.                <h1>Welcome to my personal web page!</h1>
              8.                <p>
              9.                       I am a full stack web developer in San Francisco , CA
              10.               </p>
              11.       </body>
              12. </html>

Let's disect this code a bit:
1) The first line is something that should appear on every webpage you make. <!DOCTYPE html> defines the type of
the document so the browser will render it correctly.
2) The <html> tag opens our document. Everything that shows on the webpage is placed within this tag.
3) The <head> tag defines metadata for our web page. This includes the title of the page (this is what appears in
the top of your tab or browser window). Notice that the <head> tag is indented to a new level
because the <html> tag was opened but not closed (a new idea was started but not completed).
Moreso, the <title> tag is indented another level further (the <head> tag was opened but not closed).
4) The <title> tag is closed on the same line it was opened which we usually do if the content of a tag is short.
On the other hand, the <head> tag is closed on a different line. The closing tag is indented to the same level as the matching opening one.

HTML-CSS-DOM

HTML - HyperText Markup Language - the standard markup language used to create web pages. Language of the web - has syntax rules. -Tag -elements -form a tree (DOM).
CSS - Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.
DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages).
The nodes of every document are organized in a tree structure, called the DOM tree.
So basically , Html is the language of the web.
Html has specific syntax and rules.
Instead of following general grammatical rules , we follow syntax and rules.
Basic word in html is tag. The browser turns html tags into elements that form a tree. -->DOM.
The DOM (in developer tools) is what the browser builts from an HTML document. Each html tag creates an element in the DOM that the browser uses to display the page. The document written in HTML defines the structure and content of the page. To define style we have to use CSS. CSS also uses certain syntax and rules to change how an element looks on the webpage.

THE BOX MODEL

See full box model here.
The content(blue) sits at the very centre with the texts, images , etc. and the other layers --> The padding , border and the margin protect it so that the other elements do not overlap or sit to close to the content.
The padding (green) clears the air around the content. It is the space inside the box that protects the content directly. It clears the area around the content and is affected by the background color of the box.
Border goes around the padding and the content. It is inherited from the color property of the box.
And lastly , our outermost layer is the margin which does not have a background color – it is transparent. It refers to the space between boxes.
BOX SIZING
So, HTML elements are boxes and each box has 4 components.
Because there are so many components to each box, it can often be hard to get the size of a box just right.
There are two techniques you can use to help deal with sizing issues.

1. The width of the box = (left border width +left padding width+content width+right padding width +right border width).
This makes it difficult to calculate how much space would the box actually take on the screen without recalculating it every time you change the values.
Luckily there is a way in CSS :
Easier to use model :
*{ box-sizing: border-box;}
This makes the calculation of the element size include both padding and border so that it is easier to make layouts.Just set the width of the overall box and no matter how much you tweak the padding and border size : ..the size of the box will always stay the same. Remember:Margin is not included in the box.
Now , there is a catch – this box model has been very recently developed so there are some browser specific prefixes known as Webkit , mose and MS. Just add them in front of the style definition and they are added as so:
*{-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box sizing: border-box;}
Note: Here * means universal selector. It will apply to all the elements.

2.We have been setting the size of the box in pixels. It is also possible to set it in percentage.
If you set it as the percentage, it would differ on the document window size. If you set a box to be 50% , it would take 50% area of the document window.
You can also set a max-width of the box –so it will become smaller as the document window decreased in size but not larger as you increase the size of document window.

BOX POSITIONING
Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
eg: In this concept : The box model's image and the description beside it are in a div with class="box-model".
The box-model div has been set to the rule display: flex;. The box-model div ends before the sub concept "BOX SIZING".
